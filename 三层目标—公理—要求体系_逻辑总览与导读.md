# 一、体系总图与三层定义原则

## 1.1 三层结构一览

```
T 层（5 个本质目标，与任意多智能体系统绑定）
  T1 可定义性 → T2 合法性 → T3 持续演化 → T4 开放演化 → T5 有效耦合

      ↓ 公理化（9 条，修订版）

PA 层（抽象公理，连接 T 与 G 的桥梁）
  PA1 存在性与主体—环境可区分
  PA2 有界性（所有可能无界增长的状态量）
  PA3 可恢复性
  PA4 持续再进入（再进入机会，而非仅状态变化）
  PA5 写权唯一性
  PA6 效应有终局
  PA7 信息可达性
  PA8 语义进展（目标相关进展投影 P(s)）
  PA9 观察—行动耦合

      ↓ 具体化（与粒子体系绑定）

G 层（系统目标 + 具体要求）
  Gₘ  裁剪元原则（设计判准，非系统性质）
  G0  存在性与本体结构
  G1  合法状态域（硬不变量）
  G2  单步演化完备性
  G3  持续稳定性
  G4  开放演化性
  G5  语义有效耦合     ← 与 G6 并行，不严格顺序依赖
  G6  一致性与可追溯性
  G7  认知连续性（系统特化）
  G8  工程鲁棒性
  Gf  后续优化方向     Gd  文档规范
```

## 1.2 每一层的定义原则（在回答什么）

| 层级 | 定义原则 | 一句话 |
|------|----------|--------|
| **T 层** | 本质目标层 | 任意「智能体—世界」多主体系统，若要**长期持续存在并有效运行**，必须追求哪些**本质性质**；与具体实现无关，违反则「不是跑得不好，而是不再是这类系统」。 |
| **PA 层** | 抽象公理层 | 若想保住 T1–T5，系统在**结构上至少必须满足什么**；比 T 更具体（有结构含义），比 G 更抽象（与粒子体系无关），是 T 与 G 的桥梁。 |
| **G 层** | 当前系统目标层 | PA 在本系统（**异步耦合粒子体系**）中**如何落地**；与 world/agents/meta、FieldProfile、notification_buffer 等绑定，仅对本系统成立。 |

## 1.3 层间逻辑（纵向）

- **T → PA**：为保住这些目标，系统在抽象结构上必须满足什么（公理化）。
- **PA → G**：这些公理在当前粒子体系中如何落地（具体化）。
- **裁剪方向**：任何进入 G 层的要求，必须能向上追溯到至少一条 PA，该 PA 必须能追溯到至少一个 T；无法追溯者不应进入主干。

# 二、每一层在回答什么问题（内在逻辑链）

## 2.1 T 层：五个本质目标各自在回答什么

| 目标 | 回答的问题 |
|------|------------|
| **T1 可定义性** | 系统**能不能被形式化描述**？（主体/环境/状态/演化/相互作用是否可区分、可表达。） |
| **T2 合法性** | 系统**在任意时刻是否都处于合法状态域内**？（状态值域、写边界、时间、真源、接口是否合法。） |
| **T3 持续演化** | 系统**在理想条件下能不能无限时间演化**？（资源会不会枯竭、调度会不会断、会不会沉默/堵塞/爆炸。） |
| **T4 开放演化** | 系统**动的不是空转**？（会不会收敛到吸收态、僵死态、无语义进展的病态循环；若有，能否逃离。） |
| **T5 有效耦合** | 主体**是否真实受环境影响并真实影响环境**？（不是若干自循环模块并列，而是真正相互作用。） |

**T 层内在逻辑（顺序与依赖）**：  
T1 是所有后续目标的前提 → T2 与 T3 并列（合法 + 能持续演化）→ T4 以 T3 为前提（还能动 → 动的不是空转）→ T5 依赖 T1（可区分是定义耦合方向的前提）。

## 2.2 PA 层：九条公理各自在回答什么

| 公理 | 回答的问题 |
|------|------------|
| **PA1** | 系统**有没有良定义的状态空间与主体/环境/相互作用路径的区分**？演化算子是否对任意合法状态都能给出下一步？ |
| **PA2** | 任何可能无界增长的状态量**是否有上界或等价有界控制**？ |
| **PA3** | 任何可消耗的关键资源**是否存在从耗尽态可达的严格正恢复路径**？ |
| **PA4** | 系统在任意可达状态下**是否在有限时间内能重新获得一次合法演化机会**？（再进入。） |
| **PA5** | 每个 durable 状态分量**是否都有且仅有一个直接写权限威者**？ |
| **PA6** | 任何效应**是否都在有限时间内进入终态**？（不永久悬挂、不无限重复激活。） |
| **PA7** | 有权接收某信息的主体，**该信息是否在有限时间内（且有效期内）到达**？ |
| **PA8** | 是否定义了**目标相关进展投影 P(s)**，且**无语义进展的循环是否有不依赖人工的逃离机制**？ |
| **PA9** | 每个主体**是否既能观察环境又能通过行动对环境产生净效果**？（双向成立。） |

**PA 层内在逻辑**：  
PA2–PA3 互补（有界 + 可恢复 = 资源公理）；PA4 与 PA5 正交（再进入 vs 写权）；PA7–PA9 共同构成 T5 的输入—输出双向；PA1 被 T1、T5 共用；PA6 与 PA2 在「积压」上协同；PA8 独立对应 T4。

## 2.3 G 层：各目标在回答什么（内在逻辑链）

| G 目标 | 回答的问题 |
|--------|------------|
| **Gₘ** | **什么东西能进入高层**？—— 必须能追溯到 PA（进而 T）；维度必须有行为影响；工程层不与理想主干混写；优先维度而非补丁。 |
| **G0** | **系统能不能被定义**？—— 粒子、状态空间、存储归属、真源位置是否钉死。 |
| **G1** | **每一时刻状态是否都合法**？—— 写隔离、基线不被污染、能量有界、单前景、效应终局、唯一真源等硬不变量。 |
| **G2** | **系统能不能跑一轮**？—— 单步演化是否完整（assemble → evolve → emit → apply）、合法、无死状态。 |
| **G3** | **系统能不能永远跑**？—— 调度链不断、能量有界且可恢复、记忆/通知有界、next_wakeup 不为 null 等。 |
| **G4** | **跑的方式是否有语义意义**？—— 有效进展定义、追踪、逃离机制、消息流动≠语义进展。 |
| **G5** | **系统能不能按预期耦合运转**？—— Field×Activity 建模、感知与动作正确性、义务与生命周期。 |
| **G6** | **系统能不能被解释/恢复**？—— 可重放事件、因果顺序、先写事件再物化、StagingLedger 非真状态。 |
| **G7** | **（特化）中断后工作能不能连续**？—— memory 真状态、分层访问、域隔离、ForegroundStack、resume_token。 |
| **G8** | **系统在现实中能不能稳**？—— 在非理想计算机环境下不因数值/并发/掉线/分布式异常失控。 |

**G 层内在逻辑链（顺序与并行）**：

```
Gₘ  裁剪元原则（Meta-Principle，非系统性质，是设计判准）
 │   —— 过滤什么东西能进入高层
G0  存在性与本体结构（Existence）
 │   —— 系统能不能被定义
G1  合法状态域（Hard Invariants）
 │   —— 每一时刻必须成立，违反 = 状态非法
G2  单步演化完备性（One-step Liveness）
 │   —— 系统能不能跑一轮
G3  持续稳定性（Asymptotic Stability）
 │   —— 系统能不能永远跑
G4  开放演化性
 │   —— 跑的方式是否有语义意义
G5  语义有效耦合（Functional Correctness）  ← 与 G6 并行
G6  一致性与可追溯性（Auditability）       ← 与 G5 并行，不严格依赖
 │   —— 系统能不能被解释/恢复
G7  认知连续性（系统特化）
 │   —— 中断后工作能不能连续（LLM agent 特化）
G8  工程鲁棒性（Engineering Robustness）
    —— 在非理想计算机环境下能不能稳
```

# 三、定义与跨域类比

## 3.1 核心概念在不同体系中的类比

| 本体系概念 | 定义（简要） | 物理中的类比 | 计算机体系中的类比 | 芯片/处理器体系中的类比 |
|------------|--------------|--------------|--------------------|--------------------------|
| **状态（state）** | 系统在某一时刻的完整可区分情形 | 相空间中的点、广义坐标与动量 | 进程/线程的寄存器+内存+外设状态 | 寄存器文件、Cache、流水线阶段状态 |
| **合法状态域** | 被允许的状态集合，不越界、不矛盾 | 相空间中的允许区域（约束流形） | 类型安全、不变量、协议状态机合法状态 | 合法微状态、未定义行为以外的状态 |
| **演化（evolution）** | 状态随时间的转移规则 | 动力学方程、哈密顿流 | 程序语义、状态转移函数、事件驱动更新 | 时钟驱动状态转移、指令语义 |
| **写权唯一** | 每个 durable 状态分量有唯一权威写入者 | 守恒量的唯一源、广义力的唯一施加者 | 单一写入者、ownership、不可变+显式更新 | 单端口写、写使能与地址唯一确定写入者 |
| **有界性** | 可能无界增长的量有上界或等价控制 | 守恒律、有界能量、相空间紧致 | 资源上限、配额、背压、GC | 队列深度、缓冲区上界、流控 |
| **可恢复性** | 从耗尽态存在严格正恢复路径 | 耗散系统的平衡态可达性、吸引子 | 资源回收、重启、故障恢复路径 | 电源管理中的恢复态、唤醒路径 |
| **再进入（调度）** | 有限时间内重新获得演化机会 | 周期性驱动、外部扰动注入 | 调度器、事件循环、唤醒 | 时钟、中断、唤醒信号 |
| **效应有终局** | 效应在有限时间内进入终态 | 瞬态衰减、过程有明确结束 | 请求-响应闭环、超时、终态机 | 事务提交/中止、指令退休 |
| **信息可达** | 信息在有限时间内到达有权接收者 | 信号传播、因果锥 | 消息传递、RPC 超时、送达保证 | 总线仲裁、缓存一致性协议下的可见性 |
| **进展投影 P(s)** | 目标相关的进展度量 | 李雅普诺夫函数、序参量 | 业务进度、事务提交、检查点 | 退休指令数、提交阶段进度 |

## 3.2 三层本身的类比

| 本体系 | 物理类比 | 计算机/芯片类比 |
|--------|----------|------------------|
| **T 层** | 物理定律（如守恒律、稳定性要求） | 架构原则（如一致性模型、可扩展性目标） |
| **PA 层** | 数学形式化（如李雅普诺夫条件、可控可观） | 抽象规范（如进程代数、类型系统公理） |
| **G 层** | 具体系统的设计约束与实现参数 | 具体 ISA、微架构、OS/运行时约束 |

# 四、公理层的数学化表述与必要性

## 4.1 部分公理的数学化表述

**PA3 — 可恢复性公理（Recoverability）**

- **表述（自然语言）**：任何可被消耗的关键资源，都必须存在从任意耗尽态可达的严格正恢复路径。
- **数学化**：  
  设资源 R 的取值为非负，耗尽态为 R = 0。则要求：  
  ∃ 活动/状态转移路径，使得在该路径上 ΔR > 0（严格正），且该路径从任意满足 R = 0 的可达状态出发可达。  
  形式化：  
  ∀ 可达状态 s 满足 R(s)=0，∃ 路径 π 从 s 出发，且 ∃ 步 k：R(π^k(s)) > R(s)（或更一般地，沿 π 存在严格正增量）。

**PA4 — 持续再进入公理（Scheduling Validity）**

- **表述（自然语言）**：系统在任意可达状态下，都必须在有限时间内重新获得一次合法演化机会。
- **数学化**：  
  设 S_reachable 为可达状态集，E(s, t) 为在时刻 t 对状态 s 的演化结果（或「下一时刻状态」）。则要求：  
  ∀s ∈ S_reachable，∃t' > t_now，使得在 t' 至少有一个组件发生状态转移（即系统不永远静默）。  
  形式化：  
  ∀s ∈ S_reachable，∃t' > t_now：E(s, t') ≠ s（或：存在某组件在 t' 被调度并发生转移）。

**PA2 — 有界性公理**

- **表述**：对任何可能无界增长的状态量 X，存在有限上界或等价有界控制。
- **数学化**：  
  ∀ 设计阶段识别的「可能无界增长」的状态量 X，∃ 常数 M 或等价机制，使得在任意可达轨迹上 X 被限制在有界范围内（或 X 的表示/影响有界）。

**PA6 — 效应有终局公理**

- **表述**：任何效应在有限时间内进入终态。
- **数学化**：  
  设 effect 有生命周期状态；终态集为 Terminal。则：  
  ∀ effect e 在时刻 t0 生成，∃t1 < t0 + T_max（T_max 为系统定义的有界时间），使得 e 在 t1 时状态 ∈ Terminal，且之后不再被激活。

**PA5 — 写权唯一性公理**

- **表述**：每个 durable 状态分量有且仅有一个直接写权限威者。
- **数学化**：  
  设 S_durable 为 durable 状态空间，分量分解为 s = (s₁, …, sₙ)。则 ∀i，∃ 唯一组件（权威者）A_i，使得只有 A_i 或经 A_i 的 apply 协议可修改 sᵢ；其他组件对 sᵢ 的写入路径必须经 A_i，即 ∀ 写操作 w 若修改 sᵢ，则 w 必须由 A_i 发起或经 A_i 接受。

**PA8 — 语义进展公理**

- **表述**：存在目标相关进展投影 P(s)；若在循环 C 内 P(s) 无净变化，则存在不依赖人工的逃离机制。
- **数学化**：  
  ∃ 进展投影 P : S → V（V 为有序或度量空间）。若 ∃ 可达循环 C（即轨迹进入 C 后周期重复），且 ∀s ∈ C 有 P(s) 在 C 上为常数（无净变化），则 ∃ 机制 M（由系统或环境触发，不依赖人工输入），使得从 C 中某状态出发经 M 可到达 C 外状态。即：在「无进展循环」上系统非吸收。

## 4.2 为什么必要、为什么重要（若不满足会怎样）

| 公理 | 若不满足会怎样 | 为何重要 |
|------|----------------|----------|
| **PA1** | 无法定义 agent、world、耦合；行为方向性无法定义；体系概念失去基础。 | 可定义性是讨论任何运行、正确性、稳定性的前提。 |
| **PA2** | 资源枯竭、内部积压爆炸、被历史拖死、在有限时间内丧失演化能力。 | 无界增长在有限实现中必然导致崩溃或语义失效。 |
| **PA3** | 消耗型动作趋向永久停机；即使有上界，到达下界后卡死。 | 长期运行必须避免「只耗不补」的单调走向。 |
| **PA4** | 系统可能永远沉默，变成静态系统；agent 无法响应环境。 | 调度/触发链断则系统不再演化。 |
| **PA5** | 耦合边界消失；因果不可追踪；审计无法进行；黑箱互写。 | 写权唯一是状态可解释、可审计、可复现的根基。 |
| **PA6** | 系统被历史效应拖死；无法区分效应是否完成；通知/义务无限积压。 | 效应无终局会导致状态不确定与资源耗尽。 |
| **PA7** | 主体与环境解耦；重要事件永远无法影响 agent；退化为自循环状态机。 | 信息不到达则「主体受环境影响」不成立。 |
| **PA8** | 固定点僵死；周期性空转；假活性；不断触发但无语义进展。 | 开放演化要求「动得有进展」，否则是假运行。 |
| **PA9** | agent 退化为纯被动接收器或盲目执行器；真实多主体—世界系统不存在。 | 观察与行动缺一则耦合不成立。 |

# 五、推导链与目标依赖关系

## 5.1 主推导链（从目标到要求）

- **正向**：T（要什么）→ PA（结构上至少满足什么）→ G（在本体系中如何落地）。  
- **反向（追溯）**：每条 G 要求 → 归属的 G 目标 → 具体化的 PA → 服务的 T。  

**推导链示例（自上而下）**：  
- T5（有效耦合）→ PA7（信息可达）、PA9（观察—行动）→ G5（语义有效耦合）→ 例如 G5-R009（local/external 阈值区分）、G5-R017（动作可达域由场与活动共同决定）。  
- T3（持续演化）→ PA4（持续再进入）→ G3（持续稳定性）→ G3-R003（next_wakeup 永不为 null）、G3-R001（time-driven 调度必须存在）。  

**推导链示例（自下而上追溯）**：  
- **G3-R003**（next_wakeup 永不为 null）→ G3（持续稳定性）→ PA4（持续再进入）→ T3（持续演化）。  
- **G4-R001**（有效进展定义 P(s)）→ G4（开放演化性）→ PA8（语义进展）→ T4（开放演化）。

## 5.2 目标间依赖关系（谁依赖谁）

**T 层**  
- T1 为前提；T2、T3 并列；T4 依赖 T3；T5 依赖 T1（通过 PA1 共享）。

**G 层**  
- G0 不依赖 G1–G8。  
- G1 依赖 G0。  
- G2 依赖 G0、G1。  
- G3 依赖 G2、G1。  
- G4 依赖 G3。  
- G5 依赖 G0、G2；**G5 与 G6 并行**（互不依赖）。  
- G6 依赖 G0、G1。  
- G7 依赖 G3、G6；不修改 G1。  
- G8 依赖 G0–G7。

**子集/细化关系（谁是谁的细化）**  
- G1（合法状态域）中的「写隔离」「能量有界」「效应终局」等，是 **T2（合法性）** 在粒子体系中的**细化**；同时写隔离、真源等又是 **PA5** 的**具体化**。  
- G2（单步演化完备性）是 **T3（持续演化）** 的「单步」侧在粒子体系中的**细化**，具体化 PA1、PA4、PA5。  
- G3（持续稳定性）是 **T3** 的「长时」侧的**细化**，具体化 PA2、PA3、PA4、PA6。  
- G4（开放演化性）是 **T4** 的**直接具体化**，唯一主要对应 PA8。  
- G5（语义有效耦合）是 **T5** 的**细化**，具体化 PA7、PA9（及 PA1 次要）。  
- G6（可追溯性）是 **T1/T2** 的「可重建/可审计」侧的**细化**，具体化 PA5、PA6。  
- G8 是 **PA2–PA9** 的**工程增强**，不是新 T，而是同一批 PA 在现实环境下的实现补强。

# 六、不满足时的风险与案例

## 6.1 按层汇总：不满足时可能出现的风险

| 层/目标 | 不满足时可能出现的风险 |
|---------|------------------------|
| **T1** | 根本得不到一个系统；无法讨论运行、正确性、稳定性。 |
| **T2** | 系统进入非法状态；后续演化结果不可信。 |
| **T3** | 永久沉默、资源耗尽停机、内部积压溢出、有限时间终止。 |
| **T4** | 假活性、假稳态、病态循环、形式运行但语义冻结。 |
| **T5** | 各自自循环的 agent；不可感知的环境；名义上的多主体而非真实相互作用。 |
| **PA2** | 资源枯竭、内部积压爆炸、被历史拖死。 |
| **PA3** | 资源永久耗尽，演化停止；到达下界后卡死。 |
| **PA4** | 系统永远沉默，变成静态系统。 |
| **PA5** | 耦合边界消失；因果不可追踪；审计无法进行。 |
| **PA6** | 系统被历史效应拖死；通知/义务无限积压。 |
| **PA7** | 主体与环境解耦；重要事件永远无法影响 agent。 |
| **PA8** | 固定点僵死；周期性空转；无语义进展的假活性。 |
| **PA9** | agent 退化为纯被动或盲目执行器；真实多主体—世界系统不存在。 |

## 6.2 案例：不满足 PA4（持续再进入）时的风险

**场景**：某实现中，agent 的 `next_wakeup` 在某些分支下被置为 null，且没有默认策略自动填入；调度器只调度「next_wakeup 非 null」的 agent。

**不满足**：系统在「next_wakeup 为 null」的可达状态下，永远无法重新获得演化机会（PA4 违反）。

**风险**：该 agent 永远沉默；若所有 agent 都因同样原因变为 null，则整个系统静默，不再响应环境，**T3（持续演化）** 失效。

**案例结论**：PA4 的「再进入」是「系统能不能永远跑」的关键一环；对应到 G 层即 G3-R003（next_wakeup 永不为 null）与 time/event-driven 调度必须存在。

## 6.3 案例：不满足 PA8（语义进展）时的风险

**场景**：某任务场中，agent 与用户来回发送消息，但从不执行 deliverable 提交、议程更新、义务完成等「进展动作」；系统没有定义 P(s)，也没有无进展时的逃离机制。

**不满足**：系统在「形式上有消息流动」的循环中，P(s) 无净变化，且无逃离机制（PA8 违反）。

**风险**：**T4（开放演化）** 失效——系统看起来在运行，实则语义冻结；任务永远不推进，形成假活性。若再叠加资源消耗（如每轮调用 LLM），还会加剧 **T3** 的长期压力。

**案例结论**：PA8 对应 G4（进展定义、追踪、软/强逃离）；缺少则「动的不是空转」无法保证。

# 七、谁是谁的子集、谁是谁的细化（小结）

- **T 层**：五个本质目标是「多智能体系统长期持续有效运行」的**概念分解**；彼此有前提与并列关系，但不是子集关系。  
- **PA 层**：九条公理是 T 的**结构必要条件**的分解；PA2–PA3、PA7–PA9、PA2–PA6 等有成对/协同关系，PA 之间多为正交或共享（PA1），而非严格子集。  
- **G 层**：  
  - **G0** 是「存在性」的**整体具体化**（PA1、PA5 的落地）。  
  - **G1** 是「合法状态域」的**整体具体化**（PA5、PA2、PA6 的落地），与 G0 是并列的两大块（存在性 vs 不变量）。  
  - **G2、G3** 是「持续演化」的**分阶段细化**：G2 单步，G3 长时。  
  - **G4** 是 T4 的**直接细化**（仅主要对应 PA8）。  
  - **G5、G6** 是 T5 与 T1/T2 不同侧面的**并行细化**（功能正确 vs 可追溯）。  
  - **G7** 是 T3 在 LLM agent 上的**特化细化**（认知连续性）。  
  - **G8** 是 PA2–PA9 的**工程增强**，不是新 T 的子集，而是同一批 PA 的「现实实现」补强。  
- **Gₘ** 不是系统性质，是**设计判准**，约束「什么能进入 G」；**Gf、Gd** 是后置方向与文档规范，不参与 T–PA–G 主干子集关系。
