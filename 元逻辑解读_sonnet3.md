# 异步耦合粒子系统·元逻辑解读

> 版本：sonnet3  
> 日期：2026-03-12  
> 定位：本文档不是实现规范，而是对三层目标—公理—要求体系（v2.2）内在逻辑的解读性文档。  
> 目标读者：初次接触本体系的研究者、希望理解"为什么这样设计"的协作者、沙盘推演前的预读材料。

---

## 0. 导读：这份文档解释什么

本体系由三层构成：**T 层（本质目标）→ PA 层（抽象公理）→ G 层（当前系统目标）**。

这三层不是三个平行的规范文件，而是一条**推导链**：

```
T 层  ——"任何智能体—世界系统，若要长期存在，必须追求什么"
  ↓  公理化（为了保住 T，结构上必须满足什么？）
PA 层 ——"不依赖具体实现的、最抽象的结构约束"
  ↓  具体化（在当前粒子体系中，PA 如何落地？）
G 层  ——"与粒子体系绑定的、具体的系统目标与要求"
```

**为什么需要这三层？**

| 只有规范（G 层）的问题 | 三层体系的解决方式 |
|---|---|
| 规范越来越多，不知道哪条更根本 | T 层钉死什么是"根本性质"，其他都是细化 |
| 不知道两条规范是否矛盾 | 追溯到同一 PA 公理，矛盾即在 PA 层可见 |
| 不知道能不能删减某条规范 | 看删掉后 PA 公理是否被破坏，PA 是否仍能保住 T |
| 实现细节和本体混写，读不懂 | G 层只谈系统性质，工程补丁单独成章 |

## 1. 三层架构的核心逻辑

### 1.1 层级的"定义原则"

每一层有且只有一个核心问题：

| 层级 | 核心问题 | 约束来源 |
|---|---|---|
| **T 层** | 这类系统"存在"和"有意义"的前提是什么？ | 从"多主体—世界系统"的概念本身推出 |
| **PA 层** | 为了保住 T，系统结构上至少需要什么性质？ | 从 T 层逻辑推导 |
| **G 层** | 在粒子体系的具体实现中，PA 如何落地？ | 从 PA 层 + 粒子体系定义 |

关键约束：**每条 G 层要求都必须能向上追溯到至少一条 PA 公理**。无法追溯的要求不应进入主干。这被称为"裁剪元原则"（Gₘ）。

### 1.2 层间的"向上追溯"与"向下推导"

```
向下推导（演绎）：
  "T3 持续演化" 
    → 为了保住它，需要 PA2（有界）∧ PA3（可恢复）∧ PA4（持续再进入）∧ PA6（效应有终局）
      → 在粒子体系中：G3-R003（next_wakeup 永不为 null）∧ G3-R004（能量恢复路径存在）∧ ...

向上追溯（归因）：
  G3-R003 [next_wakeup 永不为 null]
    → 服务 PA4（持续再进入）
      → 保住 T3（持续演化）
```

两个方向都成立，才算"闭环"。

### 1.3 "子集"关系图

```
T 层（最抽象，与所有多智能体系统共享）
  ├── T1 可定义性
  │     └── 是后续所有目标的前提：若系统无法被定义，则讨论运行/正确性均无意义
  ├── T2 合法性
  │     └── T1 的细化：系统能被定义之后，必须在合法状态域内变化
  ├── T3 持续演化
  │     └── T1+T2 的细化：能被定义且合法，还要能无限时间运行
  ├── T4 开放演化
  │     └── T3 的细化：能运行，还要"有意义地运行"（不能空转）
  └── T5 有效耦合
        └── T1+T3 的补充：能运行，但要求多主体间真实相互作用（不能各自自循环）

PA 层（抽象结构约束，是 T 的公理化）
  ├── PA1-PA2-PA5-PA6 主要服务 T1+T2（存在性与合法性）
  ├── PA2-PA3-PA4-PA6 主要服务 T3（持续演化）
  ├── PA8              主要服务 T4（开放演化，直接对应）
  └── PA7-PA9(+PA1)    主要服务 T5（有效耦合）

G 层（具体实现，是 PA 在粒子体系中的落地）
  └── 每个 G 目标 ⊆ 对应的一或多条 PA 公理 ⊆ 对应的一或多个 T 目标
```

## 2. T 层：本质目标

> **定义原则**：T 层描述的是"使一个系统成为'多主体—世界自治系统'的必要条件"。缺少任何一条，系统就不再是我们所定义的那类系统——不是"跑得不好"，而是"根本不是"。

### 2.1 T 层的内在逻辑链

```
T1 可定义性
  ↓ 系统首先要能被形式化表达（没有定义，后面的都无从谈起）
T2 合法性
  ↓ 定义了之后，要保证状态始终在合法域内（否则语义崩溃）
T3 持续演化
  ↓ 合法地定义，合法地运行，还要能永久运行下去
T4 开放演化
  ↓ 永久运行，但不能是无意义的空转（要有语义进展）
T5 有效耦合
  └─ 贯穿所有层：主体必须真实地与环境相互作用（否则只是独立状态机的集合）
```

**注意**：T1→T2→T3→T4 是严格的前提依赖链；T5 是并行约束，在 T1 之后始终有效。

---

### T1：可定义性

**回答的问题**：这个系统能不能被描述出来？

**最小要求**：能区分主体与环境；能描述状态；能描述演化；能描述相互作用。

**物理类比**：经典力学中，必须首先定义质点（主体）、场（环境）、力（相互作用）和运动方程（演化），才能讨论任何力学问题。缺少任一，力学体系无法建立。

**计算机类比**：操作系统中，必须首先定义进程（主体）、内存（环境/状态）、系统调用（接口/相互作用）、调度器（演化机制），才能讨论进程的行为。

**芯片类比**：必须首先定义寄存器（状态）、指令集（演化规则）、总线（相互作用路径）、时钟（时间基底），才能设计电路行为。

**若 T1 缺失**：
- 风险：根本无法描述系统，所有后续讨论都是空谈。
- 案例：一个"多智能体系统"没有定义 agent 是什么、world 是什么、它们如何交互。当你试图推演"agent A 发消息给 agent B"时，你无法描述消息从哪里来、经过什么路径、到哪里去，也无法判断这个动作是否合法。整个推演从起点就失去基础。

---

### T2：合法性

**回答的问题**：系统能不能始终保持在"有意义的状态"？

**最小要求**：状态值域合法、写边界合法、时间单调、真源唯一、接口完备。

**物理类比**：热力学第一定律——系统的能量变化必须守恒；能量不能凭空产生或消失。若违反守恒，物理模型失去可信语义。

**计算机类比**：内存安全——进程不能越界写入其他进程的内存空间。若进程 A 能任意写 B 的内存，B 的所有状态都不可信。

**芯片类比**：总线仲裁——在同一时刻，总线只能由一个主设备控制。若多个设备同时争写，数据总线上的信号叠加为噪声，所有读取都无意义。

**若 T2 缺失**：
- 风险：系统进入非法状态，后续所有演化结果不可信。
- 案例：Agent A 的 `energy` 值被其他 agent 直接写入（绕过写隔离），导致 A 的 energy 从 20 跳变为 150（超出 [0,100] 上界）。此时 energy 失去语义，任何基于 energy 的行为判断（如"是否需要休息"）都产生错误结果。后续 3 小时的推演都建立在这个错误值之上。

---

### T3：持续演化

**回答的问题**：系统能不能永远运行下去（在理想条件下）？

**最小要求**：资源不会只耗不补；累积不会只增不减；调度不会中断；演化函数不会落入死状态。

**物理类比**：一个能量守恒且有熵泵的系统——不会无限耗散能量到停止（有补充），也不会无限积累无序（有排出机制）。

**计算机类比**：垃圾回收机制——程序不能无限分配内存而不释放（否则 OOM 停机）；也需要调度器保证每个进程都能被唤醒（否则饥饿）。

**芯片类比**：时钟信号和复位机制——CPU 需要持续的时钟脉冲驱动（否则停转），需要有 watchdog 防止死锁（否则芯片挂死）。

**若 T3 缺失**：
- 风险：系统在有限时间内停止运行。
- 案例：Agent B 的 `energy` 以 -3/h 的速度消耗（工作状态），没有任何恢复机制。初始 energy = 100，33小时后 energy = 1，此时 B 被 P0 规则强制切换 recovering 状态。但如果 recovering 活动不能产生正能量增益（违反 G3-R004），则 B 的 energy 永远无法回升，B 从此进入永久沉默，系统退化为 n-1 个 agent。

**数学化表达（PA3 的直接对应）**：

$$\exists \text{ activity } a : \frac{d E}{dt}\bigg|_a > 0$$

$$\wedge \; \forall s \in S_{E=0} : s \xrightarrow{*} s' \text{ where } a \text{ is available in } s'$$

即：存在一种活动使能量严格增长，且从任意能量耗尽态，该活动都可达。

---

### T4：开放演化

**回答的问题**：系统的运行有没有语义进展（不是空转）？

**与 T3 的区别**：T3 保证"还能动"；T4 进一步要求"动的不是无意义循环"。

**物理类比**：布朗运动 vs 随机漫步——粒子一直在动（T3 成立），但如果在一个封闭势阱中永远逃不出去，就永远无法到达新的区域（T4 违反）。

**计算机类比**：活锁（livelock）——两个进程持续地相互礼让（一直在"运行"），但任何一方都无法继续推进实际工作。系统没有死亡（T3 成立），但也没有进展（T4 违反）。

**芯片类比**：流水线冲突中的气泡（bubble）——流水线一直在转（T3 成立），但由于数据相关导致大量 NOP 指令，实际有效指令吞吐量趋近于零（T4 违反）。

**若 T4 缺失**：
- 风险：假活性——系统形式上在演化，但永远不产生新的语义结果。
- 案例：Agent A 和 Agent B 由于义务循环死锁（A 等 B 先完成义务X，B 等 A 先完成义务Y），两者都处于"blocked"态。但系统每 5 分钟唤醒一次（T3 成立），A 演化后发现 B 的义务未完成，维持等待；B 同理。系统在这两个状态间机械来回，永无进展。所有参与者都在"等"，但整个工作项目永远不会推进。

**数学化表达（PA8 的对应）**：

定义进展投影函数：$P: S \to \mathbb{R}^d$（将系统状态投影到目标相关维度）

$$\text{若系统进入循环 } C \subseteq S, \; \text{且} \; \Delta P \big|_C = 0$$

$$\text{则} \; \exists \; \text{mechanism} \; M : C \xrightarrow{M} s \notin C \; \text{且} \; M \text{ 不依赖人工干预}$$

---

### T5：有效耦合

**回答的问题**：主体与环境之间有没有真实的双向交互？

**最小要求**：合法事件最终能影响相应主体（可观察）；合法动作最终能改变相应世界（可作用）；两者缺一不构成真实耦合。

**物理类比**：测量—干扰原理——量子力学中，观测会影响系统（观察侧），系统状态影响观测结果（作用侧）。若主体只能"看"不能"改"，或只能"改"不知"看到什么"，则不构成真实的量子测量。

**计算机类比**：I/O 系统——CPU 必须能读取输入设备（键盘/网络）的数据（观察侧），也必须能向输出设备（显示器/磁盘）写入结果（作用侧）。单向的 I/O 不构成完整的计算循环。

**芯片类比**：传感—控制回路——传感器读取环境（观察），执行器修改环境（行动）。若缺少任一，控制回路断裂，系统退化为开环系统。

**若 T5 缺失**：
- 风险：各 agent 成为孤立状态机，"多主体系统"名存实亡。
- 案例：设计者错误地把所有 agent 的 memory 设计为独立私有，且没有共享的 InteractionContext。Agent A 发布了一个消息，但这个消息没有任何路由机制到达 Agent B。B 永远无法感知 A 的行动，A 的动作永远不会影响 B 的状态。两者各自独立演化，与"多智能体协作"的目标完全背离。

---

## 3. PA 层：抽象公理

> **定义原则**：PA 层是 T 层的公理化——"为了保住 T1–T5，系统结构上至少需要什么"。PA 层比 T 层更具体（有结构含义），但比 G 层更抽象（不与粒子体系绑定）。

### 3.1 PA 层的内在逻辑图

```
             T1(可定义)  T2(合法)  T3(持续)  T4(开放)  T5(耦合)
                │          │          │          │          │
PA1 存在性与可区分 ●          ○          ○          ○          ●
PA2 有界性         ○          ●          ●          ○          ○
PA3 可恢复性       ○          ○          ●          ○          ○
PA4 持续再进入     ○          ○          ●          ○          ●
PA5 写权唯一性     ●          ●          ○          ○          ○
PA6 效应有终局     ○          ●          ●          ○          ○
PA7 信息可达性     ○          ○          ○          ○          ●
PA8 语义进展       ○          ○          ○          ●          ○
PA9 观察—行动耦合  ●          ○          ○          ○          ●

● = 主要服务  ○ = 次要或无直接关系
```

**每条 PA 公理回答的问题**：

| 公理 | 回答的核心问题 |
|---|---|
| PA1 | 系统能不能被形式化定义（有没有"主体"和"环境"的结构区分）？|
| PA2 | 所有可能无界增长的量，有没有上界？|
| PA3 | 被消耗的关键资源，有没有恢复路径？|
| PA4 | 系统在任意状态，在有限时间内能不能重新获得演化机会？|
| PA5 | 每个 durable 状态，有没有且只有一个权威写者？|
| PA6 | 任何生成的效应，在有限时间内能不能进入终态？|
| PA7 | 应该到达某主体的信息，能不能在有效期内到达？|
| PA8 | 若系统陷入无语义进展的循环，有没有自主逃离机制？|
| PA9 | 每个主体既能观察环境又能作用于环境吗？|

### 3.2 PA 层的"互补性"结构

PA 层的 9 条公理不是简单并列，而是形成几组互补对：

```
资源管理互补对：
  PA2（有界，防上溢/爆炸）↔ PA3（可恢复，防下溢/耗尽）
  → 两者共同保证资源处于有界有恢复的健康区间

效应生命周期互补对：
  PA6（效应有终局，防永挂）↔ PA7（信息可达，防永失）
  → PA6 防止效应永远挂起，PA7 防止信息永远到不了

耦合双向互补对：
  PA7（观察侧：信息到达）↔ PA9（行动侧：行动有效）
  → 两者共同构成"真实耦合"的双向闭环

运行活性互补对：
  PA4（持续再进入，保证有演化机会）↔ PA8（语义进展，保证机会不被空转）
  → PA4 防止"永远没机会"，PA8 防止"机会全是空转"
```

### 3.3 每条公理的数学化表达

---

**PA1：存在性与主体—环境可区分**

$$\exists (S_A, S_B, \mathcal{I}) : S = S_A \times S_B, \quad \mathcal{I} \subseteq S_A \times S_B \to \text{Events}$$

$$\text{且} \quad \forall s \in S : \delta(s) \in S \quad \text{（演化函数全定义）}$$

即：状态空间可分为"环境侧"与"主体侧"，存在明确定义的相互作用路径 $\mathcal{I}$，且演化函数在任意合法状态上有定义。

**若违反**：无法定义 agent 和 world，整个体系概念失去基础。整个文档的所有讨论都无法进行。

---

**PA2：有界性**

$$\forall X \in \{E_{energy}, |\text{notifications}|, |\text{memory}|, |\text{obligations}|, ...\} : X \leq \text{Bound}(X) < \infty$$

即：所有可能无界增长的状态量，都存在有限上界。

**若违反**（以 notification 为例）：
- 一个高流量的论坛场中，每次有人发帖都产生通知，但没有 TTL 清理机制。
- 1000 条消息后，notification_buffer 有 1000 × n 条通知（n = agent 数量）。
- assemble 时系统试图将所有通知注入上下文，context window 溢出，系统 OOM。

---

**PA3：可恢复性**

$$\exists \; \text{state sequence} \; \sigma : s_0 \xrightarrow{\sigma} s^* \quad \text{where} \quad \frac{dR}{dt}\bigg|_{s^*} > 0$$

$$\wedge \quad \forall s \in S_{R=R_{\min}} : s \xrightarrow{*} s^*$$

即：存在使关键资源 $R$ 严格增长的状态 $s^*$，且从资源最小值状态可达 $s^*$。

**物理类比**：一个弹簧—阻尼系统，弹簧提供恢复力（可恢复性），阻尼提供耗散（有消耗）。系统必须保证弹簧力永远大于零（从任意压缩态都有恢复力）。

**若违反**：agent 的 energy 耗尽后，若没有任何活动能产生正 Δenergy，则 energy 永远停在 0，agent 永久停止演化。

---

**PA4：持续再进入**

$$\forall s \in S_{\text{reachable}}, \exists t' > t_{\text{now}} : s \xrightarrow{*} \text{EvolveOpportunity}(t')$$

即：从任意可达状态，在有限时间内，系统必然获得至少一次合法演化机会（无论是时间驱动、事件驱动还是内部调度）。

**注意**：PA4 不要求演化后一定产生语义进展（那是 PA8 的职责）；只要求系统不永久失去演化机会。

**计算机类比**：操作系统调度器——无论进程当前在做什么，调度器必须保证它在有限时间内获得一次 CPU 时间片。即使进程在睡眠，也会被定时唤醒检查。

**若违反**：Agent 设置了 `next_wakeup = null`（或遗漏设置），系统的 time-driven 调度链在该 agent 处断裂。Agent 陷入永久沉默，即使有大量未处理的通知，也没有任何机制唤醒它。

---

**PA5：写权唯一性**

$$\forall s_i \in S_{\text{durable}} : |\{\text{authoritative writer of } s_i\}| = 1$$

$$\text{且} \quad \forall j \neq \text{auth}(i) : j \text{ 不能直接修改 } s_i, \text{ 只能通过 apply 路径间接影响}$$

**芯片类比**：寄存器文件（Register File）——每个寄存器只有一个写端口（单写者）。若允许多个 ALU 同时写同一寄存器，结果是未定义的竞争（race condition）。

**若违反**：
- Agent A 和 Agent B 同时持有 InteractionContext X 的写权限。
- A 把 X 的 `status` 写为 "closed"，B 同时把 X 的 `status` 写为 "active"。
- 最终 X.status 的值取决于写入的时序，不可预测。所有依赖 X.status 的后续逻辑（如 closeCondition 检查）都产生不确定结果。

---

**PA6：效应有终局**

$$\forall e \in \text{Effects} : \exists t_\infty < \infty : \text{state}(e, t_\infty) \in \text{Terminal}$$

$$\text{Terminal} = \{consumed, expired, merged, archived, rejected, cancelled, ...\}$$

即：任何生成的效应对象（通知、义务、请求、消息等），都必须在有限时间内进入终态。

**物理类比**：衰变粒子——任何不稳定粒子（效应）都有有限的平均寿命，最终衰变为稳定产物（终态）。若粒子可以无限期不稳定存在，则粒子计数器会无限溢出。

**若违反**：义务对象 O（type=respond）被创建但没有 TTL，也没有 deadline，Agent B 因为网络问题 3 天没有响应。义务 O 持续悬挂在 `active_obligations` 中。3 天后，积累了 10,000 个同类悬挂义务。系统的义务管理模块在每次 assemble 时都要扫描所有悬挂义务，O(n) 的开销使系统性能崩溃。

---

**PA7：信息可达性**

$$\forall e \in \text{Events}, \forall B_i \in \text{Recipients}(e) :$$
$$\exists t^* : t_{\text{now}} < t^* < t_{\text{expiry}}(e) \land e \in \text{AssembleInput}(B_i, t^*)$$

即：任何有权接收事件 $e$ 的主体 $B_i$，$e$ 必须在其有效期内到达 $B_i$ 的演化输入。

**若违反**：
- Agent A 在群聊场中发布了一条重要的任务分配消息（高 priority）。
- 由于通知路由逻辑错误，该消息对应的 notification 没有被投递到 Agent B 的 notification_buffer。
- Agent B 永远不知道自己有一项任务，任务永远无法被接受或拒绝。整个协作流程在 B 这里静默失败。

---

**PA8：语义进展**

定义进展投影 $P: S \to \mathbb{R}^d$（目标相关维度）

$$\text{若} \; \exists C \subseteq S : \text{system trajectory} \subseteq C \land \Delta P\big|_C = 0$$

$$\text{则} \; \exists \; M \text{（逃离机制）}: \text{Prob}[\text{system exits } C \text{ via } M] > 0$$

$$\text{且} \; M \text{ 不依赖 human intervention}$$

即：若系统陷入语义空转循环，必须存在自主逃离机制（如 N-tick 超时逃生）。

**若违反**：两个 agent 陷入义务循环死锁（A 等 B，B 等 A）。两者的 `next_wakeup` 正常更新（T3/PA4 成立），每隔 5 分钟各自演化一次，但演化后检测到另一方未完成义务，继续等待。没有任何超时机制打破这个循环。系统在这两个状态间无限来回，永远不产生任何语义进展。

---

**PA9：观察—行动耦合**

$$\forall B_i \in \text{Agents} :$$
$$\exists f^{\text{obs}}_i : S_A \to \text{Percepts}_i \quad \text{（观察：主体能看到环境）}$$
$$\land \; \exists f^{\text{act}}_i : S_{B_i} \times \text{Actions} \to \Delta S_A \quad \text{（行动：主体能改变环境）}$$

$$\text{且} \; f^{\text{act}}_i \text{ 的值域非退化（即行动确实能产生 } S_A \text{ 的净变化）}$$

**若违反（行动侧退化）**：
- Agent B 声称发出了一条消息（action = post_message），但由于 emit 路径的错误，该消息事件没有被路由到 World A 的 R_A^apply。
- World A 的 interaction_contexts 从未更新，消息从未进入任何 context 的 messages 列表。
- B 的行动产生了 intent（durable StagingLedger 有记录），但对世界没有任何实际效果。
- 系统形式上运行，但 B 的所有"行动"都是无效的，T5 违反。

## 4. G 层：当前系统目标

> **定义原则**：G 层是 PA 层公理在"异步耦合粒子系统"中的具体落地。G 层的每条要求都：  
> 1. 与粒子体系（world/agents/meta、FieldProfile、notification_buffer 等）绑定；  
> 2. 能向上追溯到至少一条 PA 公理。

### 4.1 G 层的内在目标链

```
Gₘ  裁剪元原则（Meta-Principle，非系统性质，是设计判准）
     "凡进入 G 层的要求，必须能追溯到 PA 公理"
     "优先增加维度而非补丁规则"
 │
 ↓ 回答：这个系统能不能被定义？
G0  存在性与本体结构
     ── 粒子系统由哪些部件构成？各部件持有什么？
     ── 对应：PA1（存在性与可区分）、PA5（写权唯一，真源钉死）
 │
 ↓ 回答：这个系统在任何时刻是否合法？
G1  合法状态域（硬不变量）
     ── 哪些性质在"任意时刻"必须成立？（能量有界、写隔离、通知终局、真源唯一等）
     ── 对应：PA2（有界）、PA5（写权）、PA6（效应终局）
 │
 ↓ 回答：这个系统能不能完成一轮完整的演化？
G2  单步演化完备性
     ── 四阶段（assemble→evolve→emit→apply）必须完整且合法
     ── 动作必须端到端闭环（有 ActionReceipt 或等价失败回执）
     ── 对应：PA1（全函数演化）、PA4（再进入—单步）、PA5（写权唯一—emit）
 │
 ↓ 回答：这个系统能不能永久运行？
G3  持续稳定性
     ── 调度链不断（next_wakeup 永不为 null）
     ── 能量恢复路径存在、记忆空间有界、通知空间有界
     ── 对应：PA2（有界）、PA3（可恢复）、PA4（持续再进入）、PA6（效应终局）
 │
 ↓ 回答：这个系统的运行有没有语义进展？
G4  开放演化性
     ── 进展投影 P(s) 必须有定义
     ── N-tick 超时逃生机制必须存在
     ── 对应：PA8（语义进展）
 │
 ↓ 回答：这个系统的主体和环境是否真实相互作用？
G5  语义有效耦合
     ── 场参数化（FieldProfile 9维）、活动门控（ActivityProfile 7维）
     ── 注入门槛（canInject/canWakeup）、动作合法性（allowedActions）
     ── 对应：PA7（信息可达）、PA9（观察—行动耦合）
 │
 ↓ 回答：系统发生的一切能不能被解释、审计、恢复？
G6  一致性与可追溯性
     ── Global Event Log 唯一真源、WriteEvent-first、拓扑排序
     ── 对应：PA5（写权唯一）+ G1（不变量）联合保障
 │
 ↓ 回答：agent 能不能在多轮演化中保持连续的认知？
G7  认知连续性（系统特化）
     ── Memory Store + source_field_id 域隔离 + ForegroundStack resume_token
     ── 是 G5（耦合）和 G6（追溯）在 agent 主观认知层的特化
 │
 ↓ 回答：在现实的非理想计算机环境中，系统能不能稳定运行？
G8  工程鲁棒性
     ── Hysteresis Lock、N-tick 逃生、Split-Brain 防护、系统重启恢复等
     ── 注意：G8 是工程补丁，删掉后 G0–G7 的逻辑结构不应坍塌
```

```
G0 ⊆ PA1+PA5   （本体定义是存在性和写权的具体落地）
G1 ⊆ PA2+PA5+PA6  （不变量是有界性、写权、效应终局的具体落地）
```

## 7. 裁剪判准（Gₘ）的使用方法

Gₘ 是本体系的"元原则"，不描述系统性质，而是给出"什么可以进入规范、什么必须被删除"的判准。

### 7.1 三个核心判准

**判准 1：可追溯性**

> 任何进入 G 层的要求，必须能向上追溯到至少一条 PA 公理。

```
使用方式：
  候选要求 R："agent 不应在深夜主动发起高优先级通知"
  → 能追溯到 PA 公理吗？
    PA4（持续再进入）? → 不直接相关
    PA7（信息可达）? → 相关，但 PA7 说信息应该到达，这条说信息不应发出
    PA9（观察-行动）? → 不直接相关
  → 无法找到直接的 PA 公理支撑
  → 结论：这是一条应用层/业务层的规则，不应进入 G 层主干
  → 处理：放入"应用层配置"，由 FieldProfile.cognitive_demand 和 Activity.mode 参数化表达
```

**判准 2：维度优先**

> 当需要表达复杂性时，优先通过增加参数维度解决，而不是叠加 if-else 补丁规则。

```
反面示例（补丁规则）：
  规则 A："如果 attention_impact=locked 且 task_binding=mission_locked，则 energy_cost = -4/h"
  规则 B："如果 attention_impact=engaged，则 energy_cost = -1.5/h"
  → 两条规则引用不同维度，逻辑交叉，容易产生冲突

正面示例（维度优先）：
  新增维度 cognitive_demand = minimal | moderate | high | intensive
  → cognitive_demand=intensive 则 fieldEnergyModifier = -4/h
  → 单一维度驱动，无歧义
  → 这正是 v6 新增 cognitive_demand 维度的理由
```

**判准 3：工程层分离**

> G8（工程鲁棒性）的要求不应与 G0–G7 混写。删掉 G8，G0–G7 的逻辑结构不坍塌。

```
使用方式：
  候选要求 R："系统崩溃重启后，next_wakeup 应使用 jitter 分散唤醒时间"
  → 这是"现实系统崩溃"这个非理想条件下的处理，在理想系统中不需要
  → 属于 G8（工程鲁棒性），不应写入 G3（持续稳定性）
  → 理由：若把它写入 G3，则 G3 的定义就变成"在有崩溃风险的现实中"，
           而不是"在理想条件下"，两者混淆使 G3 的语义不清晰
```

## 8. 多类比参考

### 8.1 物理系统类比

| 本体系概念 | 物理类比 | 物理学意义 |
|---|---|---|
| T1 可定义性 | 描述量子系统需要希尔伯特空间 | 没有希尔伯特空间，量子力学无从谈起 |
| T2 合法性 | 守恒律（能量/动量/角动量守恒）| 违反守恒律的系统不是物理系统 |
| T3 持续演化 | 封闭系统的热力学第二定律 + 开放系统的熵泵 | 纯耗散系统最终死亡；需要熵泵才能持续 |
| T4 开放演化 | 远离平衡态的耗散结构 | 在平衡态（最大熵）附近的系统不产生新的有序结构 |
| T5 有效耦合 | 量子纠缠 / 经典力的相互作用 | 没有相互作用，粒子是孤立的，不构成"系统" |
| PA2 有界性 | 热力学极限下的状态密度有界 | 能量密度不能无限增长 |
| PA3 可恢复性 | 弹性恢复力（胡克定律）| 被压缩的弹簧必须能弹回；纯耗散无弹性恢复则死亡 |
| PA5 写权唯一 | 波函数归一化条件（测量坍缩到单一本征态）| 量子测量后系统处于确定状态，不存在叠加写权 |
| PA8 语义进展 | 李雅普诺夫稳定性 vs 渐近稳定性 | 不渐进收敛到有意义吸引子的系统只是形式演化 |

### 8.2 计算机系统类比

| 本体系概念 | 计算机系统类比 | 说明 |
|---|---|---|
| T1 可定义性 | 程序必须有类型系统和语义 | 无类型的数据不能被有意义地操作 |

| T5 有效耦合 | I/O 系统（输入+输出均合法有效）| 单向 I/O 不构成完整交互 |
| PA1 存在性 | 进程描述符（PCB）| 没有 PCB，OS 无法管理进程 |
| PA2 有界性 | 内存配额 + GC 垃圾回收 | 无上界的内存分配导致 OOM |
| PA4 持续再进入 | 调度器时间片轮转 + 定时中断 | 没有定时中断，进程可能永久占用 CPU |
| PA5 写权唯一 | 互斥锁（mutex）| 多线程无锁写同一变量：race condition |
| PA6 效应有终局 | 文件描述符必须关闭 / future 必须 resolve | 永久挂起的 future 导致资源泄漏 |
| PA8 语义进展 | 死锁检测 + 强制重试 / WDT（watchdog timer）| 活锁/死锁检测机制 |
| G8 工程鲁棒性 | 容错机制（retry、circuit breaker、checkpoint）| 在非理想环境（网络延迟、进程崩溃）下的补强 |

### 8.3 芯片设计类比

| 本体系概念 | 芯片/硬件类比 | 说明 |
|---|---|---|
| T1 可定义性 | 必须有指令集架构（ISA）| 没有 ISA，电路无法被设计成计算机 |
| T2 合法性 | 时序约束（setup/hold time）| 违反时序约束导致亚稳态，输出不可信 |
| T3 持续演化 | 时钟信号 + 复位机制 | 没有时钟，电路停转；没有复位，死锁后无法恢复 |
| T4 开放演化 | 流水线效率（避免气泡）| 流水线一直在转，但全是气泡（NOP），有效指令吞吐为零 |
| T5 有效耦合 | 传感—执行器回路（感知+控制）| 传感器无输出 or 执行器无响应：开环系统 |
| PA1 存在性 | 寄存器文件 + 总线定义 | 没有寄存器，无法存储状态；没有总线，无法传输数据 |
| PA2 有界性 | 寄存器位宽 + 溢出检测 | 32位寄存器溢出→未定义行为；需要有界控制 |
| PA4 持续再进入 | WDT（Watchdog Timer）| WDT 定期检查系统是否还在运行，超时则复位 |
| PA5 写权唯一 | 总线仲裁器（bus arbiter）| 同一时刻只有一个主设备能占用总线写数据 |
| PA6 效应有终局 | DMA 传输完成中断 | DMA 传输必须有完成通知，否则 CPU 无法知道何时可以使用数据 |
| G8 工程鲁棒性 | ECC 内存 + 热冗余 | 在有噪声/电磁干扰/温度变化的真实环境下的补强 |

## 9. 快速使用指南

### 9.1 如何判断一条新规则应该放在哪里

```
Step 1：能追溯到 PA 公理吗？
  否 → 不属于 G 层主干，考虑放入业务配置或文档附录
  是 → 继续 Step 2

Step 2：是理想条件下的要求，还是工程补强？
  理想条件下 → G0~G7（根据追溯到的 PA 公理选择对应 G）
  工程补强 → G8

Step 3：能用维度取值表达吗？
  是 → 用维度（如 cognitive_demand、interrupt_tolerance 的取值）而非 if-else 规则
  否 → 写成 G 层要求，附带 PA 根源标注

Step 4：检查是否已有覆盖
  查对应 G 目标下是否已有等价要求
  若有 → 补充说明而非新增，避免规范重复
  若无 → 新增，格式：[简称] 正文 → PA根源：PAx → 次目标：Gy → 层级
```

### 9.2 如何验证系统是否满足某条目标

以 G3-R003（next_wakeup 永不为 null）为例：

```
验证问题："在任意时刻，任意 agent 的 next_wakeup 是否都是有效时间戳？"

验证路径：
  1. 检查 R_B^apply 中是否有如下逻辑：
     "若 Output 未提供 next_wakeup，则系统自动填入 defaultNextWakeup(...)"
  2. 检查 defaultNextWakeup 是否有 MIN_WAKEUP_INTERVAL 保护
     "next_wakeup = max(computed, now + MIN_WAKEUP_INTERVAL_SEC)"
  3. 检查系统重启恢复流程（onSystemRestart）是否恢复所有过期的 next_wakeup
  
若以上任一缺失 → G3-R003 未满足 → PA4 被违反 → T3 存在风险
```

### 9.3 如何进行沙盘推演的前置检查

在开始沙盘推演之前，确认以下前提：

```
✅ G0 已满足：知道系统由哪些粒子构成（V_A、V_B、V_C）
✅ G1 已满足：了解所有硬不变量（energy 有界、写隔离、通知终局等）
✅ G2 已满足：理解四阶段演化（assemble→evolve→emit→apply）
✅ G5 已满足：理解 canInject/canWakeup/allowedActions 的门控机制
✅ 活动预设表已掌握：知道每个 preset 对应的 7 维参数
✅ FieldProfile 已掌握：知道每个场的 9 维参数及其对系统的影响

推演时的核心检查清单（六问）：
  Q1. 这个事件能到达主体吗？（canInject 是否为 true？）
  Q2. 到达后会被看到吗？（通知优先级 >= effective_threshold？）
  Q3. 是否会打断当前活动？（canWakeup？ interrupt_tolerance？）
  Q4. 若响应，输出什么动作？（allowedActions 中是否包含？）
  Q5. 动作如何写回状态？（R_B^apply 中的写回路径是什么？）
  Q6. 写回后，下次何时演化？（next_wakeup 更新到什么值？）
```

## 10. 附录：关键术语对照表

| 术语 | 在 T 层的含义 | 在 PA 层的含义 | 在 G 层的含义 |
|---|---|---|---|
| 状态 | 系统当前的所有性质 | 形式化状态空间中的点 | V_A × V_B₁ × ... × V_Bₙ（durable）|
| 演化 | 系统从当前状态到下一状态的转变 | 全定义演化算子 δ: S→S | assemble→evolve→emit→apply 四阶段 |
| 耦合 | 主体与环境的真实相互作用 | 观察函数 f_obs + 行动函数 f_act | FieldProfile × canInject × allowedActions |
| 有界 | 不能无限增长 | ∀X: X ≤ Bound(X) < ∞ | energy ∈[0,100]，notification TTL，memory 分层 |
| 终局 | 效应最终进入确定状态 | ∀e: ∃t: state(e,t) ∈ Terminal | Notification: consumed/expired/merged/archived |
| 真源 | 状态的权威来源 | 写权唯一者 auth(sᵢ) | Global Event Log + 粒子本体变量 |
| 进展 | 系统语义上向目标靠近 | ΔP(s) > 0（P 为进展投影）| 义务完成、议程推进、任务交付 |
| 可恢复 | 被消耗的资源有补充路径 | ∃σ: s_{R=0} →^* s* where dR/dt|_{s*} > 0 | energy_profile=recovering 活动必须存在 |
