# 《当前体系的全量沙盘任务库 v2（重新导出）》

> 说明：本次为重新导出版。上一版下载失败后，已在新路径下重新生成。

## 0. 文档定位

本文档把三份方法模板与当前 `T–PA–G` 体系直接绑定，并把此前 **320 个统一沙盘场景** 全部回收入主任务库；同时补入 **160 个按 `T / PA / G` 缺口反推的新任务**。因此，本版不再是“首版框架任务库”，而是面向首轮系统性验证的**全量任务库**。

本版任务总数：**480 个**。其中：
- 新增框架/桥接/体系任务：**160**
- 继承旧场景任务：**320**
- GEN：**115**
- MAP：**30**
- SYS（新增体系任务）：**15**
- LEGACY（继承场景，归入 SYS 域）：**320**
- 一般/桥接/体系绑定/场景级合计优先级：P0=301，P1=179，P2=0

## 1. 分类原则

### 1.1 一级分类
- **GEN**：一般性判据任务，主要服务于 `T / PA` 层，尽量可跨体系复用。
- **MAP**：桥接任务，负责检查通用判据如何映射到当前体系。
- **SYS**：当前体系新增结构任务，直接验证对象、状态机、路由、写回与长期运行机制。
- **LEGACY**：此前已做过的 320 个正常/极端场景，现已全部按 `T / PA / G` 主映射回收入库。

### 1.2 二级分类

- 一般性任务：定义性、合法性、有界性、恢复性、终局性、进展性、耦合性、跨层一致性、体系校验。
- 桥接任务：对象映射、真源映射、effect 映射、进展映射、工程层与理想层拆分、关键对象映射。
- 体系任务：上下文、前景栈、本地场、能量/睡眠、通知、义务、写回、租约、路由、工程故障、过载、持久化、灾害、多智能体、进展、长期运行、治理。
- 继承场景：基线矩阵、通知风暴、写回与崩溃、义务与死锁、前景场、网络与时间、灾害与世界级风险、长期运行、本地物理冲突、并发与权限等。

## 3. 使用方法

1. **先跑 GEN 与 MAP 的 P0 任务**，确认体系本身、映射关系与真源/终局/进展判据无根本断裂。

2. **再跑 SYS 的 P0 任务**，验证当前体系主干机制：Context、ForegroundStack、Notification、Obligation、WriteEvent-first、route/apply、hazard、replay。

3. **最后回收 LEGACY 320 场景**，把原有正常/极端沙盘全部挂回新的 `T / PA / G` 分类体系。

4. 对所有 `P3=概率性不满足` 与 `P4=完全不满足` 的场景，必须按模板生成根因追溯链。

## 4. 字段说明

- `任务ID`：统一唯一编号。`GEN/MAP/SYS` 为新增任务；`LEGACY-*` 为回收的 320 场景。
- `主验证层`：该任务主判定落点。一般性任务常在 `T/PA`，体系任务常在 `G`。
- `T映射 / PA映射 / G映射`：该任务的主要上收路径，不代表唯一相关项。
- `任务性质`：一般、桥接、体系绑定、正常、极端。
- `失败优先追因`：首轮下钻建议，不替代最终根因追溯表。
