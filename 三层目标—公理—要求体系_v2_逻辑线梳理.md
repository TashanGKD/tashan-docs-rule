# 《三层目标—公理—要求体系 v2》逻辑线梳理

> 本文档对 `三层目标—公理—要求体系_v2.md` 中的**每一条逻辑线**进行梳理，包括：纵向（层与层之间）、横向（同层内各维度之间）、以及其它显式或隐式的逻辑关系。  
> 目的：显式化「不止三层」的完整逻辑结构，便于校验、演进与实现对照。

---

# 一、纵向逻辑线（层与层之间）

## 1.1 主追溯方向：G → PA → T

**文档定义**（§ 文档结构与使用说明）：

- **逻辑方向**：每条具体要求 → 归属的 G 目标 → 具体化的 PA 公理 → 服务的 T 目标。
- **裁剪方向**：任何进入 G 层的要求，必须能向上追溯到至少一条 PA 公理，该公理必须能追溯到至少一个 T 目标。无法追溯者不应进入主干。

**逻辑线含义**：

- 每条 **Gx-Rxxx** 通过「PA根源」指向若干 **PAy**，再通过 T→PA 映射指向若干 **Tz**。
- 形成链：`Gx-Rxxx → Gx → PAy → Tz`。这是文档的**主逻辑线**，保证「没有无根的要求」。

## 1.2 反向推导方向：T → PA → G

**文档定义**（§ 第三部分、第五部分）：

- **T → PA**：为保住 T1–T5，结构上至少需满足的抽象公理（PA1–PA9）。
- **PA → G**：这些公理在当前粒子体系中如何落地为 G0–G8 等目标及具体要求。

**具体映射**（纵向线一览）：

| T 目标 | 必要公理（T→PA） | 主要落地的 G（PA→G） |
|--------|------------------|----------------------|
| T1 可定义性 | PA1, PA5 | G0（本体、真源表）, G1（写隔离、真源）, G2（全函数演化）, G6（可重建） |
| T2 合法性 | PA2, PA5, PA6 | G1（写隔离、有界、效应终局）, G6（事件顺序、先写事件再物化） |
| T3 持续演化 | PA2, PA3, PA4, PA6 | G2（四阶段、再进入）, G3（调度链、能量恢复、有界）, G7（记忆、中断恢复） |
| T4 开放演化 | PA8 | G4（进展定义、追踪、逃离、消息≠进展） |
| T5 有效耦合 | PA7, PA9, PA1（次要） | G0（共享世界）, G2（assemble 可见）, G5（Field×Activity、感知与动作） |

**注意**：PA1 同时服务 T1 与 T5，形成「可定义性」与「有效耦合」的共享前提（可区分主体/环境），文档明确说明此为共享依赖而非循环。

## 1.3 理想层与工程层的纵向分离

- **G0–G7**：理想闭环，均能追溯到 PA 与 T。
- **G8**：对 PA2–PA9 的**现实/工程增强**，不是新的抽象公理；删掉 G8 后 G0–G7 的逻辑结构不应坍塌（Gm-R003）。
- 因此存在两条纵向「束」：**T–PA–G0~G7**（理想）与 **T–PA–G0~G7 + G8**（含工程实现）。

# 二、T 层内部逻辑线（横向）

## 2.1 T1 作为前提

- **表述**：T1 是所有后续目标的前提；若 T1 不成立，无法讨论运行、正确性、稳定性或实现。
- **逻辑线**：T1 → T2, T3, T4, T5（概念上，可定义性是讨论合法性、演化、开放演化、耦合的前提）。

## 2.2 T2 与 T3 的并列与互补

- **T2**：任意时刻处于合法状态域（状态不越界、不矛盾）。
- **T3**：理想条件下持续无限时间演化（不耗尽、不沉默、不堵塞、不爆炸）。
- **关系**：合法是演化的前提之一（非法状态则演化无意义）；演化是「长期存在」的体现。二者并列，共同约束「系统可长期运行且状态可信」。

## 2.3 T3 与 T4 的递进

- **文档**：T4 明确「与 T3 的区别」：T3 保证「还能动」，T4 保证「动的不是空转」。
- **逻辑线**：T4 以 T3 为前提；在「持续演化」成立的基础上，再约束「无语义进展的收敛/循环」必须可逃离。

## 2.4 T5 与 T1 的依赖

- **文档**：PA1 服务 T1 与 T5；T5 的「有效耦合」需要「主体—环境可区分」（否则无法定义耦合方向）。
- **逻辑线**：T5（有效耦合）在概念上依赖 T1（可定义性）提供的区分基础；在公理层体现为 PA1 同时服务 T1 与 T5。

## 2.5 T 层内部小结

- **顺序关系**：T1 为前提；T2、T3 并列；T4 在 T3 之上；T5 依赖 T1。
- **无**：T2 与 T4、T5 之间无文档内显式横向约束；T 层不定义 T 之间的「实现依赖」，仅定义「为何是本质目标」与「违反后果」。

# 三、PA 层内部逻辑线（横向）

## 3.1 PA2 与 PA3：资源公理的一对

- **表述**：PA3 与 PA2 互补；PA2 要求有上界（不能无限增长），PA3 要求有恢复路径（不能单调耗尽）；两者合起来构成长时稳定的资源公理。
- **逻辑线**：PA2（有界性）+ PA3（可恢复性）→ 共同支撑 T3（持续演化）中的「资源不枯竭」维度。

## 3.2 PA4 与 PA5 的区分

- **表述**：PA4 不要求状态必须发生变化，只要求系统不能永久失去再次演化的机会；PA5 约束写权唯一。
- **逻辑线**：PA4 管「再进入机会」（调度/触发链不断），PA5 管「谁可以写谁」（状态权威与因果）；二者正交，共同参与 T3（PA4 还参与 T5 的「agent 可响应环境」）。

## 3.3 PA7 与 PA9：T5 的「输入—输出」双向

- **表述**：PA7 是 T5 有效耦合的「输入侧」保证（信息可达）；PA9 是「双向」保证（观察 + 行动），二者形成闭环。
- **逻辑线**：PA7（信息到达）+ PA9（行动有效）→ 共同定义 T5「有效耦合」的充分侧；缺一则不成立真实多主体—世界系统。

## 3.4 PA1 的共享角色

- **表述**：PA1 服务 T1（可定义性）与 T5（有效耦合）；T1 的「可区分」是 T5「耦合方向性」的前提。
- **逻辑线**：PA1 在 PA 层内不依赖其他 PA，但被 T1 与 T5 共用，是跨 T 维度的共享前提。

## 3.5 PA6 与 PA2 在「积压」上的交叠

- **PA2**：可积累状态（如通知、队列）需有界控制。
- **PA6**：效应须在有限时间内进入终态，不得永久悬挂。
- **逻辑线**：PA6 从「效应生命周期」约束积压；PA2 从「状态量有界」约束积压；二者在「不因积压破坏 T2/T3」上协同，但侧重点不同（终局 vs 上界）。

## 3.6 PA8 的独立性与对 T4 的覆盖

- **表述**：T4 仅由 PA8 主要覆盖；PA8 是非病态轨道的直接条件，无其他公理能覆盖语义进展要求。
- **逻辑线**：PA8 在 PA 层内独立承担「开放演化」的公理化，与 PA2–PA7、PA9 无文档内显式横向依赖。

## 3.7 PA 层内部小结

- **成对/协同**：PA2–PA3（资源）；PA7–PA9（耦合双向）；PA2–PA6（积压与终局）。
- **共享**：PA1 被 T1、T5 共用。
- **正交/独立**：PA4（再进入）与 PA5（写权）；PA8 独立对应 T4。

# 四、G 层内部逻辑线（横向）

## 4.1 G 目标之间的依赖顺序（附录 B.1）

文档显式给出：

- **G0** 不依赖 G1–G8（最底层存在性定义）。
- **G1** 不变量在 G0 存在性成立的基础上定义。
- **G2** 依赖 G0（粒子三元组结构）和 G1（合法状态保证演化有意义）。
- **G3** 依赖 G2（能演化一步才能谈持续）和 G1（不变量保证稳定性条件不被污染）。
- **G4** 依赖 G3（系统先活着，才能谈是否空转）；G4 不依赖 G5/G6/G7。
- **G5** 依赖 G0（Field/Activity 存储位置）和 G2（emit 路由）；**G5 与 G6 并行**（非顺序依赖）。
- **G6** 依赖 G0（Event Log 归属）和 G1（写权唯一保证审计可信）。
- **G7** 依赖 G3（记忆有界）和 G6（可追溯支持恢复）；G7 不修改 G1 的定义。
- **G8** 依赖 G0–G7（工程实现层，不修改上层定义）。

**逻辑线归纳**：

```
G0（存在性）
  ├→ G1（合法状态域）
  ├→ G2（单步演化）[ 依赖 G0, G1 ]
  ├→ G6（可追溯性）[ 依赖 G0, G1 ]
  └→ G5（有效耦合）[ 依赖 G0, G2 ]

G2, G1 → G3（持续稳定性）
G3 → G4（开放演化性）
G3, G6 → G7（认知连续性，特化）
G0–G7 → G8（工程鲁棒性）
```

## 4.2 G5 与 G6 的并行关系（附录 B.3）

- **表述**：系统可以满足 G5（功能正确）但不满足 G6（不可追溯），反之亦可；两者均依赖 G0 与 G1/G3，但互相不依赖。
- **逻辑线**：G5 与 G6 是两条**平行的横向目标**，不是前后顺序；实现上可分别满足或同时满足。

## 4.3 G4 与 G8-R009 的「理想—工程」分工

- **G4**：要求存在进展定义、追踪与逃离机制（理想闭环）。
- **G8-R009**：N-tick 超时逃生是 G4 的工程实现之一；G4 的删除会去掉语义进展要求，G8-R009 的删除只削弱实现韧性。
- **逻辑线**：G4（理想）→ G8-R009（工程实现），同一根纵向线在 G 层拆成「要求」与「实现」两层。

## 4.4 「次目标」形成的 G 内横向服务关系

大量 Gx-Rxxx 标注了**次目标：Gy**，表示该条要求除服务本目标 Gx 外，还**顺带服务** Gy。例如：

- G0 多条 → 次目标 G2、G5、G6（存在性/真源为演化、耦合、可追溯提供基础）。
- G1-R005 → 次目标 T3（基线不被污染，保护调度恢复能力）。
- G2-R002 → 次目标 G5（assemble 可见上下文）；G2-R007 → 次目标 Gₘ（六问判准）。
- G3-R006 → 次目标 G7（记忆有界支撑认知连续性）；G4-R002 → 次目标 G8（N-tick 依赖 last_progress_at）。
- G5 多条 → 次目标 T3、G4、T2（感知/动作/生命周期与演化、开放演化、合法性交叉）。

**逻辑线**：这些「次目标」在 G 层内部形成**横向服务网**，不是严格层级，而是「一条要求多目标服务」的关系。

## 4.5 Gₘ、Gf、Gd 与主干 G0–G8 的关系

- **Gₘ**：判准/设计原则，约束「什么能进入 G 层」「维度必须有行为影响」「工程层不混入理想主干」「优先维度而非补丁」。不直接对应 PA/T，但 Gm-R001 要求任何 G 层要求须能追溯到 PA（进而 T）。
- **Gf**：长期质量与可演化方向，可后置；缺少时 G0–G8 仍可满足。
- **Gd**：文档规范元要求，约束主文档写法与结构，不参与 T–PA–G 追溯链。

**逻辑线**：Gₘ 是 G 层的「元规则」；Gf、Gd 是旁支（后置或文档层），不参与纵向主干。

## 4.6 G7 的「特化」与边界

- **表述**：G7 是 LLM-based agent 系统的特化目标；建立在 G3、G6 基础上，但不是其前提；缺少 G7 仍可满足 G0–G6；G7 的实现不应反过来影响 G1 的定义。
- **逻辑线**：G7 在依赖关系上是 G3、G6 的**下游**，在适用范围内是**可选特化**；与 G1 的关系是「不修改 G1」，而非「依赖 G1」。

## 4.7 最小闭环核（G3-R013）中的 G 内汇总

- **表述**：最小闭环核必须包含：粒子三元组、S_total 定义、写隔离（G1-R001）、四阶段演化（G2-R001）、持续再进入（G3-R003）、唯一真源（G1-R012）、energy bounded（G1-R007 + G3-R004）、memory bounded（G3-R006）、notification terminal（G1-R010 + G3-R008）；当前额外必要：单前景场（G1-R008）、P0 生存规则、Field×Activity 组合（G5）。
- **逻辑线**：G3-R013 把 **G0（结构）、G1（不变量）、G2（演化）、G3（稳定性）、G5（耦合）** 中的关键项打成一条「最小闭环」横向线，用于验证 T3 在本系统中的充要实现。

# 五、其它逻辑线（跨层或特殊类型）

## 5.1 层级属性与适用范围

- **T 层**：对所有多智能体系统成立，与粒子体系无关。
- **PA 层**：对所有多智能体系统成立（抽象公理），与粒子体系无关。
- **G 层**：仅对本系统成立，与粒子体系绑定。

**逻辑线**：T、PA 是「通用层」；G 是「本系统实例层」。纵向推导时，通用层不变，实例层可随体系演进调整（如 Gf 后置、G8 工程增强）。

## 5.2 要求层级与类型（硬要求 / 硬不变量 / 禁止项 / 判准 / 设计原则 / 增强项）

- **硬要求、硬不变量**：必须满足，否则违反对应 T/PA/G。
- **禁止项**：明确不允许的实现方式。
- **判准**：用于判断某规则/维度是否可进入高层（如 G2-R007 六问、G3-R012 充要条件）。
- **设计原则**：优先选择（如 Gm-R004 增加维度而非补丁）。
- **增强项**：推荐或可选，缺了不破坏 G0–G8 完备性。
- **声明性约束**：如 G7-R006 声明 G7 的边界。

**逻辑线**：同一 G 目标下，不同「层级」的要求在严格性上形成一条**强度轴**（从必须到可选），与纵向追溯链正交。

## 5.3 完备性与最小充分性

- **T 层完备性**：T1–T5 覆盖「任意多智能体系统持续有效运转」所需的本质维度。
- **PA 层完备性**：每个 T 均有充分 PA 覆盖；PA1–PA9 为 T 的最小充分集（文档 3.2 逻辑校验）。
- **G 层完备性**：每条 PA 在 G 层至少有 one 具体化；每个 G 目标能追溯到至少一条 PA。

**逻辑线**：三条「完备性」线分别约束 T、PA、G 自身无缺口，且层间映射满射/有源。

## 5.4 裁剪与入口控制

- **裁剪方向**（见 1.1）：只有能追溯到 PA（进而 T）的要求才应进入主干。
- **Gm-R001**：对任何进入 G 层的要求，必须能给出其服务哪条 PA 公理（进而哪个 T 目标）的理由。
- **G2-R007（六问判准）**：演化规则至少能回答六问之一，否则不应进入高层。

**逻辑线**：「裁剪」与「判准」是从**设计过程**约束逻辑一致性的线：所有进入者必须有据可查、可回答关键问题。
