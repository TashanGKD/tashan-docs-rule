# 目标—要求体系 v1.1

本文档先给出完整目标树，再给出按目标树重排后的要求总表。它是一个独立的新文档，不再引用旧版编号体系作为正文内容。

## 一、分级目标树

### M0 最小化目标
只保留对“理想情况下最小无限时间演化”必要的对象、要求与机制。

### G0 存在性目标
系统能被形式化描述；若 G0 不成立，则无法谈论演化、闭环、稳定性与实现。

### G1 硬不变量目标
系统在任意时刻、任意演化步骤后都必须成立的条件；违反即进入非法状态。

### G2 稳定性目标
系统在理想条件下可无限时间演化；这是最小生存线。

### G3 活性目标
合法事件最终能到达该到达的地方；合法动作最终能被执行并产生效果。

### G4 一致性与可追溯性目标
状态变更不产生矛盾、双写或不可重建历史。

### G5 语义正确性目标
系统行为必须按设计语义发生。

### G6 认知连续性目标
每个 agent 在多轮演化中能保持自我连续感与工作连续感。

### G7 鲁棒性目标
在非理想条件下，系统稳健降级而不是崩溃。

### G8 质量/公平/可演化目标
长期运行中维持质量、公平与后续演化空间。

## 二、目标依赖关系

```text
M0 最小化目标
└── G0 存在性
    └── G1 硬不变量
        └── G2 稳定性
            ├── G3 活性
            ├── G4 一致性与可追溯性
            ├── G5 语义正确性
            └── G6 认知连续性
                └── G7 鲁棒性
                    └── G8 质量 / 公平 / 可演化
```

## 三、三档标签

- **硬要求**：理想情况下要让系统成立、闭环、长期演化，必须满足。
- **可后置**：不属于最小主干，但对语义完整性、认知连续性、质量或后续演化重要，可后置。
- **工程增强**：实现层 / 非理想条件下的防故障、稳态化与恢复增强。

# M0 最小化目标

只保留对“理想情况下最小无限时间演化”必要的对象、要求与机制。

## M0.1 最小化与失格判准

### M0-001 [硬要求]
若删掉某个对象、字段或规则后，不破坏闭环、不破坏长期稳定、不破坏恢复能力，则该对象不应占据高层。
- 父要求：`M0.1`
- 可代码化：`否`
- 建议表达：`governance`

### M0-002 [硬要求]
任何高层规则至少应能回答：到达、看见、中断、输出、写回、再调度六问之一。
- 父要求：`M0.1`
- 可代码化：`否`
- 建议表达：`governance`

### M0-003 [硬要求]
若一个维度不能影响输入、输出、状态更新或调度，它不应进入第一性原理层。
- 父要求：`M0.1`
- 可代码化：`否`
- 建议表达：`governance`

### M0-004 [硬要求]
最小闭环核至少应包含：粒子三元组、`S_total`、写隔离、四阶段演化、再进入机制、唯一真源、energy bounded、memory bounded、notification terminal。
- 父要求：`M0.1`
- 可代码化：`否`
- 建议表达：`governance`

### M0-005 [硬要求]
单前景场假设、P0 生存规则、Field × Activity 组合，是当前最小闭环核的重要组成。
- 父要求：`M0.1`
- 可代码化：`否`
- 建议表达：`governance`

### M0-006 [硬要求]
如果一个系统会因为 context、通知或记忆无限积压而在有限时间内失效，则它不满足第一性原理要求。
- 父要求：`M0.1`
- 可代码化：`否`
- 建议表达：`governance`

### M0-007 [硬要求]
如果一个系统会因为注意力基线被场/活动永久污染而失去调度恢复能力，则它不满足长期演化要求。 --- ## N. 工程实现阶段仍建议满足的要求（但不属于理想主干）
- 父要求：`M0.1`
- 可代码化：`否`
- 建议表达：`governance`
