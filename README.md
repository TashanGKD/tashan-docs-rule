## 🔬 背景：为什么需要一个公理化底座

现有 LLM 多智能体社会模拟研究面临三个公认瓶颈（JASSS 综述, Li & Tao 2026）：

1. **缺乏长期世界级闭环**——多数工作更像"场景型模拟"，难以长期自主运行
2. **基础设施不是理论变量**——环境、制度与调度常被视为工程细节，而非被纳入形式化
3. **验证停留在均值拟合**——忽视异质性、方差与边界，系统复杂度不等于科学可信度

> **本研究的核心判断**：若缺少严格的形式化、验证和主张分层，系统再复杂也难以上升为科学方法。数字世界必须先被**公理化定义**，才能被**物理实现**，才能被**计算验证**，才能让真实人以**数字分身**进入。

本仓库的 T-PA-G 三层体系正是这一"可公理化数字世界底座"的第一层：**公理规格层**。

## 💡 核心创新点

### 创新一：首套跨语境数字世界公理规格体系

现有多智能体系统研究中，尚无一套同时能被**数学、物理与计算机**三种语境理解的通用数字世界定义。本体系打通了这三个语境：

| 语境 | 本体系的对应 |
|---|---|
| **数学** | 为多主体系统定义状态空间、演化算子、约束条件与进展投影（参照 Laubenbacher 等将 ABM 嵌入有限动力系统的工作）|
| **物理** | 定义开放耦合动力系统的健康性——持续演化（不耗散堵塞）、开放演化（不假活不空转）、有效耦合（不解耦退化）|
| **计算机** | 构造"社会型状态机"的形式化规格：写隔离、真源唯一、接口完备、持续再进入、效应有终局（对应 TLA+ 传统的安全性与活性）|

### 创新二：严格的三层分离架构（T / PA / G）

```
T 层（本质目标层）—— 对任何多智能体系统成立，与实现无关
     ↓ 公理化推导
PA 层（抽象公理层）—— T 与 G 之间的结构性桥梁，与粒子体系无关
     ↓ 具体化落地
G 层（当前系统目标层）—— 与当前粒子架构（world/agents/meta）绑定，含完整子要求
```

**这一分离的意义**：T 层与 PA 层可跨系统、跨版本复用；G 层被允许与实现绑定。任何工程改动若冲击 T/PA 层，说明系统的本质性质发生了变化，而非仅是实现细节的调整。这种分层在多智能体系统规格领域目前尚无先例。

### 创新三：语义进展公理（PA8 / T4）——超越普通软件正确性

T4（开放演化）与 PA8（语义进展）的组合，要求系统不仅要"还能运行"（T3 持续演化），还要"运行不是空转"——即定义并追踪目标相关的进展投影 P(s)，在无进展循环中提供可自主触发的逃离机制。

这使本体系明显超出传统软件工程的正确性讨论，进入了**语义活性层**，直接对应多智能体系统中的活锁检测、义务循环僵死与 LLM 假活性问题。

### 创新四：可复用的沙盘验证方法

配套规范文档（《三层体系元逻辑说明书 v2》）将验证方法形式化为"沙盘推演"：

- 不是普通测试，而是以 T-PA-G 为唯一基准，在具体状态场景中进行显式因果推导
- 要求每条失败都能沿 G→PA→T 链条向上收敛到确定性根因
- 允许发现**理论本身的遗漏**——若无处归因则说明公理体系需要增量修正

这一方法兼具**系统规范验证**与**理论元验证**双重功能，在 ABM 验证研究传统中属于新的推进。

### 创新五：为"真人进入数字世界"定义接口

本公理体系明确规定了数字分身（Digital Twin）进入世界所需满足的结构性条件：主体状态（V_B）的写权唯一性、认知连续性（G7）的记忆域隔离、义务系统与进展追踪。这为后续的"真人如何进入数字世界"研究（0→1 构建、1→100 逼近、分层评测）提供了明确的公理层接口，而非仅是工程约定。

## 🌐 可视化功能详解

### 三列关系图谱（主视图 `viz/index.html`）

三层架构直观呈现，各列独立滚动，SVG 连线层覆盖全局：

```
┌── T 列（蓝色区域）──┬── PA 列（紫色区域）──┬── G 列（绿色区域，双列）──┐
│  每节点显示：        │  按5组聚类：          │  每节点显示：              │
│  · 定义（1行）       │  定义组/边界组/        │  · PA根源 + T目标 chip     │
│  · 最小要求（3-4条） │  生存组/进展组/        │  · 前4条子要求（默认展开） │
│  · 违反后果          │  耦合组               │  · 每条：ID+级别标签+标题  │
│  · [→案例·类比]按钮  │  + PA四对互补对        │  · [展开更多] 按钮         │
└──────────────────────┴───────────────────────┴───────────────────────────┘
```

**交互：**
- 悬停节点 → SVG 连线高亮（T→PA 蓝色，PA→G 紫色），无关节点淡至 20%
- 点击节点 → 弹窗：定义 / 数学表达式 / ⚠ 违反案例 / 跨域类比（物理·计算机·芯片）

### 有向图谱（`viz/viz-graph.html`）

全部 23 个节点一屏可见，悬浮矩形 + 阴影，58 条有向边五类分色：

| 边类型 | 含义 | 数量 |
|---|---|---|
| T 层内部依赖 | T1→T2→T3→T4，T1→T5（串行前置依赖） | 4 |
| T→PA 公理化 | 为保住每个 T 目标，必须满足哪些 PA 公理 | 14 |
| PA 互补对（双向） | PA2↔PA3 / PA6↔PA7 / PA7↔PA9 / PA4↔PA8 | 8 |
| PA→G 具体化 | 每条 PA 在当前粒子体系中如何落地 | 23 |
| G 层内部依赖 | 串行链 + 并行 + 特化关系 | 10 |

**交互：** 悬停节点 → 相关节点放大 1.1×，连线高亮至 88%，**每条激活连线旁显示完整逻辑说明文字**，右侧面板列出该节点的全部相关边

### 英文版（`viz/viz-en.html`）

与中文主视图完全对应，英文名称优先（来自 M0 双语术语字典），中文为小字副语言，UI 全英文。
